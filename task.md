Типи даних та схема
Isar підтримує такі примітивні типи та їх List-варіанти [web:schema]:

Тип	Розмір	Діапазон
byte	1 байт	0–255
short	4 байти	±2,147,483,647
int	8 байт	±9.2e18
float	4 байти	±3.4e38
double	8 байт	±1.7e308
bool	-	true/false/null
String	-	UTF-8
DateTime	-	мікросекундна точність (мс у браузері)
Також підтримуються @embedded-об'єкти (вкладені без обмежень по глибині) та enum з чотирма стратегіями серіалізації: ordinal, ordinal32, name, value [web:schema]. Важливий нюанс: числові типи не мають окремого null-представлення - замість цього використовуються sentinel-значення (int.MIN, double.NaN) [web:schema].

Індекси
Це одна з найсильніших сторін Isar. Три типи індексів:

IndexType.value - стандартний B-tree індекс, підходить для рівності та range-запитів

IndexType.hash - хеш-індекс для рядків, ефективніший по пам'яті, але тільки для точного збігу

IndexType.hashElements - хеш для кожного елементу списку

Складові (composite) індекси - повністю підтримуються, і саме вони дають той "a lot faster" у бенчмарках. Multi-entry індекси дозволяють індексувати кожен елемент масиву окремо - це основа для full-text search.

CRUD, транзакції, запити
Всі операції доступні через IsarCollection:

Читання: синхронне (getSync) та асинхронне (get/findAll)

Запис: тільки через writeTxn() - транзакція обов'язкова

ACID: автоматичний rollback при помилці

Query Builder: типобезпечний DSL з and(), or(), xor(), filter(), where(), limit(), offset(), sortBy()

Watchers: реактивні Stream<void> і Stream<List<T>> на колекцію, об'єкт або запит - lazy та non-lazy варіанти

Зв'язки між колекціями
Isar підтримує реляційні зв'язки через IsarLink<T> і IsarLinks<T>:

IsarLink<T> - відношення 1:1

IsarLinks<T> - відношення 1:N та N:N

@Backlink() - автогенерований зворотний зв'язок без додаткового коду

Ключова деталь: всі зв'язки - lazy. Ти мусиш явно викликати .load() і .save(), інакше value буде порожнім. На non-web платформах є автозавантаження при першому зверненні, але покладатися на це в продакшні - ризик.

Міграції та схема
Міграція схеми - автоматична і zero-cost для простих змін (додавання поля, перейменування через @Name). Забута міграція при зміні структури без @Name - краш при відкритті бази. Версіонування схеми задається вручну через параметр schema.

Інструментарій
Isar Inspector - вбудований дебаг-інструмент, що запускається при flutter run --debug. Дозволяє в реальному часі переглядати колекції, виконувати запити, редагувати дані і перемикатися між інстансами. Для unit-тестів поза Flutter потрібно викликати Isar.initializeIsarCore(download: true), який автоматично завантажить потрібний бінарник [ pub.dev].
